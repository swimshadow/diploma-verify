# Система верификации подлинности дипломов

Платформа для выдачи, хранения и проверки подлинности дипломов о высшем образовании. Объединяет цифровую подпись (ЭЦП), блокчейн, OCR-распознавание и QR-сертификаты в единую экосистему с разграничением ролей: ВУЗ, студент, работодатель, администратор.

---

## Реализованная функциональность

- **Регистрация и аутентификация** — JWT + refresh-токены, разделение по ролям (student / university / employer / admin)
- **Электронная цифровая подпись (ЭЦП)** — ВУЗ подписывает дипломы RSA-2048; challenge-response верификация подписи
- **Загрузка и управление дипломами** — ВУЗ загружает PDF-дипломы с метаданными (ФИО, серия, номер, степень, специальность, дата выдачи)
- **OCR-распознавание дипломов** — автоматическое извлечение данных из PDF и изображений (Tesseract, русский + английский)
- **ML-классификация подлинности** — endpoint для подключения модели классификации (заготовка с интерфейсом)
- **Блокчейн** — append-only цепочка блоков, SHA-256 хеширование, фиксация каждого верифицированного диплома
- **QR-сертификаты** — генерация уникального QR-кода для каждого верифицированного диплома; мгновенная проверка сканированием
- **Сквозное шифрование** — AES-256-GCM шифрование payload'ов между клиентом и сервером
- **Шифрование ПДн** — персональные данные (ФИО) хранятся в зашифрованном виде (Fernet) + хеш для поиска
- **Уведомления** — email-рассылка и in-app уведомления о статусах дипломов
- **Административная панель** — управление пользователями, модерация дипломов
- **Запросы на верификацию** — работодатель подаёт запрос на проверку диплома, студент/ВУЗ подтверждает
- **Мультиплатформенность** — единый Flutter-клиент для Web, Android, iOS, Desktop
- **Demo-режим** — автоматическое создание тестовых аккаунтов, дипломов и QR-сертификатов

---

## Особенность проекта

1. **Микросервисная архитектура** — 14 независимых сервисов за единым API-шлюзом (Caddy), каждый со своим healthcheck и возможностью горизонтального масштабирования
2. **Тройная защита подлинности** — диплом фиксируется одновременно цифровой подписью ВУЗа (RSA-2048), записью в блокчейн (SHA-256 chain) и QR-сертификатом
3. **Сквозное шифрование трафика** — все запросы между клиентом и сервером шифруются AES-256-GCM поверх HTTPS
4. **AI-пайплайн** — OCR-извлечение данных из скана диплома + подготовленный интерфейс для ML-классификатора подлинности
5. **Мгновенная проверка по QR** — работодатель сканирует QR-код на сертификате и получает результат верификации в реальном времени

## Основной стек технологий

| Слой | Технологии |
|------|-----------|
| **Фронтенд** | Flutter 3.11, Dart, flutter_bloc, go_router, dio, mobile_scanner, encrypt |
| **Бэкенд** | Python, FastAPI, SQLAlchemy, Pydantic |
| **ML / AI** | Node.js, Tesseract.js (OCR), pdf-parse |
| **База данных** | PostgreSQL 16 |
| **Кеширование** | Redis 7 |
| **Блокчейн** | Собственная реализация (SHA-256, append-only chain) |
| **Криптография** | RSA-2048 (ЭЦП), AES-256-GCM (payload), Fernet (ПДн), bcrypt (пароли) |
| **Инфраструктура** | Docker, Docker Compose, Caddy (reverse-proxy) |

## Установка

### Быстрый запуск (один скрипт)

```bash
git clone <repository-url>
cd diploma-verify
chmod +x deploy.sh
./deploy.sh
```

Скрипт автоматически:
1. Соберёт Flutter Web Release
2. Скопирует сборку в `release/`
3. Создаст `backend/.env` из `.env.example` (если отсутствует)
4. Соберёт и запустит все Docker-контейнеры

### Ручной запуск

```bash
# 1. Сборка фронтенда
cd frontend
flutter build web --release --no-tree-shake-icons
cd ..

# 2. Подготовка .env
cp backend/.env.example backend/.env
# Отредактировать backend/.env при необходимости

# 3. Запуск бэкенда
cd backend
docker compose build
docker compose up -d
```

### Доступ после запуска

| Ресурс | URL |
|--------|-----|
| Веб-приложение | `http://localhost:8000` |
| По локальной сети | `http://<IP>:8000` |
| Swagger API | `http://localhost:8000/docs` |
